Tour de bases de datos: instalación y cómo se ven por dentro

Las doce opciones mencionadas en la auditoría, agrupadas por cómo se instalan (servidor local, un solo binario, o solo una cuenta en la nube). Cada una incluye pasos reales de instalación y una recreación fiel de su interfaz — no son capturas de pantalla literales (las interfaces cambian de versión en versión), pero sí reproducen con precisión dónde está cada cosa.

Por dónde empezar según lo que ya decidimos

No hace falta instalar las doce. Según la auditoría, estas son las que realmente importan para tu situación:

  • MySQL/MariaDB — ya la tenés corriendo en XAMPP; el tour es para que conozcas phpMyAdmin a fondo, ya que hoy es tu única ventana a la base de datos.
  • PostgreSQL + pgAdmin — si algún día seguís el Combo 07 o 09 (Supabase/FastAPI), o simplemente querés comparar motores en tu propia PC.
  • PocketBase — para probar el Combo 08 en un fin de semana, tal como sugiere la recomendación final.
  • Supabase y Neon — para probar el Combo 07 (BaaS gestionado) sin instalar nada, solo con una cuenta gratuita.

El resto (MongoDB, SQLite, Firebase, Appwrite, Convex, Turso, Cloudflare D1) están igual de completos abajo, por si te interesa explorarlos o simplemente entender por qué quedaron descartados en la auditoría.

01

MySQL / MariaDB

Ya la tenés

Servidor local clásico. La administrás con phpMyAdmin, que ya viene dentro de tu XAMPP.

phpMyAdmin
Panel web de administración incluido en XAMPP
localhost/phpmyadmin

Cómo abrirla (ya instalada)

  1. Con Apache y MySQL corriendo en el panel de XAMPP, abrí el navegador.
  2. Entrá a http://localhost/phpmyadmin
  3. En la columna izquierda vas a ver alquileres_db — hacé clic para desplegar sus tablas.

02

PostgreSQL + pgAdmin

Instalación nueva

El motor relacional más rico en funciones del grupo. Se administra con pgAdmin 4, un panel muy similar en espíritu a phpMyAdmin pero más denso en opciones.

pgAdmin 4
Se instala junto con PostgreSQL o por separado
app de escritorio

Instalación en Windows

  1. Descargá el instalador desde postgresql.org/download/windows (incluye pgAdmin 4 por defecto).
  2. Corré el .exe, elegí puerto (5432 por defecto) y contraseña del usuario postgres.
  3. Al terminar, se abre pgAdmin desde el menú inicio — te pide esa misma contraseña para conectar al "Server" local.

03

MongoDB + Compass

Descartada en la auditoría

Se incluye por completitud — tu dominio es relacional, no de documentos, así que no aplica aquí. Igual vale conocer cómo se ve, porque es muy distinta a las anteriores.

MongoDB Compass
Cliente gráfico oficial, se instala junto con MongoDB Community
app de escritorio

Instalación en Windows

  1. Descargá el instalador .msi desde mongodb.com/try/download/community.
  2. Elegí setup "Complete" y marcá la casilla "Install MongoDB Compass" (viene tildada por defecto).
  3. Al terminar, Compass abre solo y te ofrece conectar a mongodb://localhost:27017.

Cómo se ve

La diferencia visual que importa: no hay filas/columnas fijas como en phpMyAdmin/pgAdmin — cada "documento" es un bloque tipo JSON que puede tener campos distintos entre sí. Cómodo para datos irregulares, incómodo quirúrgico para algo como "el total de un cobro debe sumar exactamente sus 6 conceptos", que en SQL se garantiza con una simple relación.

SQLite + DB Browser

Un solo archivo

No es un servidor — es un archivo .db en tu disco. Lo abrís con un visor liviano.

DB Browser for SQLite
Visor/editor gratuito, sin instalar ningún servidor
app portable

Instalación

  1. Descargá desde sqlitebrowser.org (hay versión portable, sin instalador).
  2. Abrí la app → "Open Database" → elegí cualquier archivo .sqlite/.db.
  3. Si no tenés uno, "New Database" crea un archivo vacío al instante.

05

PocketBase

Combo 08 de la auditoría

Un ejecutable de ~15MB que es servidor + base de datos + panel admin, todo junto. El más rápido de "instalar" de toda la lista: no instala nada, solo lo ejecutás.

Panel admin de PocketBase
Se abre solo en el navegador al iniciar el binario
127.0.0.1:8090/_/

Instalación (Windows)

  1. Descargá el .zip para Windows desde pocketbase.io/docs — es un solo pocketbase.exe.
  2. Descomprimilo en una carpeta y abrí una terminal ahí.
  3. Corré: ./pocketbase.exe serve
  4. Abrí http://127.0.0.1:8090/_/ — la primera vez te pide crear la cuenta admin (email + contraseña).

Lo más distinto: "Collections" en la barra izquierda son tus tablas, pero se crean con un formulario visual (nombre de campo + tipo: texto, número, relación, archivo) en vez de escribir CREATE TABLE. La pestaña "Auth" ya te da usuarios/login funcionando sin escribir una línea de código — es la pieza que hoy no tenés en absoluto.

06

Supabase

Combo 07 de la auditoría

Nada que instalar en tu PC: creás cuenta gratis y trabajás desde el navegador (Supabase Studio). Por dentro corre PostgreSQL real.

Supabase Studio
Dashboard web de tu proyecto
supabase.com/dashboard

Cómo empezar

  1. Entrá a supabase.com → "Start your project" → login con GitHub.
  2. "New Project" → elegí nombre, contraseña de base y región (elegí una cercana a Perú, ej. São Paulo).
  3. Esperá ~2 minutos mientras aprovisiona el proyecto — te lleva directo al Studio.

07

Neon

Reemplazo vigente de PlanetScale

PostgreSQL "serverless" con una idea distintiva: podés crear una copia completa de tu base (una "branch") en segundos, como una rama de git.

Neon Console
Dashboard web + SQL Editor integrado
console.neon.tech

Cómo empezar

  1. Entrá a neon.tech → "Sign up" (GitHub/Google).
  2. "Create a project" → nombre + versión de Postgres (17 recomendada).
  3. Te da la cadena de conexión al toque; el SQL Editor ya está listo para usar sin instalar cliente alguno.

Firebase

Descartada en la auditoría

La consola de Google para Firestore (su base NoSQL). Se incluye por completitud — el costo por uso es menos predecible que las demás BaaS.

Firebase Console
Panel web de Google Cloud
console.firebase.google.com

Cómo empezar

  1. Entrá con una cuenta Google a console.firebase.google.com.
  2. "Add project" → nombre → Google genera un ID único.
  3. En el menú "Build" → "Firestore Database" → "Create database" → elegí ubicación.

Appwrite

Cloud o self-hosted

Similar en espíritu a Supabase/Firebase, con la ventaja de poder auto-hospedarlo gratis en tu propio VPS sin límites artificiales.

Appwrite Console
Cloud (appwrite.io) o self-hosted vía Docker
cloud.appwrite.io

Self-hosted en un VPS (Docker)

  1. docker run -it --rm --volume /var/run/docker.sock:/var/run/docker.sock --volume "$(pwd)"/appwrite:/usr/src/code/appwrite:rw --entrypoint="install" appwrite/appwrite:1.9.5
  2. El instalador web corre en el puerto 20080; esperá ~30s y abrí http://tu-servidor.
  3. Wizard de 4 pasos: dominio, base de datos (MariaDB o MongoDB), y creación de la cuenta admin.

Convex

Sin capa gratuita duradera

Base de datos "reactiva" pensada para apps en tiempo real. Se incluye por completitud del panorama BaaS.

Convex Dashboard
Panel web, deploy vía CLI
dashboard.convex.dev

Cómo empezar

  2. El CLI abre el navegador para loguearte y crear el deployment.
  3. npx convex dev sincroniza tu esquema/funciones y abre el dashboard.

11

Turso

Vigente, más CLI que dashboard

SQLite "distribuido" — a diferencia de las demás BaaS, la experiencia principal es por terminal, no por panel web.

Turso CLI
Panel web más simple; lo fuerte es la terminal
turso db shell

Cómo empezar

  1. curl -sSfL https://get.tur.so/install.sh | bash
  2. turso auth signup (o login si ya tenés cuenta).
  3. turso db create alquileres → turso db shell alquileres te abre un prompt SQL directo.

Cómo se ve

terminal — turso db shell alquileres
alquileres> SELECT codigo_unidad, tarifa_alquiler_base FROM unidades WHERE codigo_unidad='302'; codigo_unidad tarifa_alquiler_base 302 600.0 alquileres> _

La diferencia más honesta de toda la lista: no hay una grilla clickeable tipo Supabase — es una terminal SQL directa (o clientes de terceros no oficiales como DB Pro). Encaja mejor si ya te sentís cómodo en línea de comandos que si preferís todo con mouse.

Cloudflare D1

No corre PHP — solo por completitud

SQLite dentro del ecosistema Cloudflare Workers. Se administra por CLI (Wrangler) con un panel web de apoyo.

Wrangler CLI + Cloudflare Dashboard
CLI principal, dashboard como complemento visual
wrangler d1

Cómo empezar

  1. npm install -g wrangler → wrangler login.
  2. wrangler d1 create alquileres-db — te da un ID para tu wrangler.toml.
  3. También podés crearla sin CLI: dashboard de Cloudflare → Workers & Pages → D1 → "Create Database".
